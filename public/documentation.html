Movies RESTful API

CRUD API I'm building using node.js and mongoDB that allow users access information about different movies, directors, and genres.
Also users will be able to authentificate their profile and sign in, update their personal information and create a list of their own favorite movies.
(N.B This API is proveded with security mesures like: encrypted Passwords, CORS logic preventig cross-origin requests and JTW tolken authentifications)

All features of this API:

  • View information about different movies (such as date of release, director, actors, movie poster among others)
  • View bio of movie directors
  • View informations about different movie genres
  • Create account via signing in
  • Update or edit profile info (Such as (Username, Password, Email, Birthday)
  • Create “Favorite Movies” list and edit in case want to add or remove certain movie title
  • Delete profile
  • All users data are

This API has the following endpoints:

End point request HTTP method Description
/movies GET Returns a list of All movies in the API
/movies/:Title GET Returns data about a single movie
/movies/:Title/Genre GET Returns data about a genre
/directors/:Name GET Returns data about a director
/documentation GET Serves this documentation file
/users POST Allows new users to be registered. (The request body object must contain all fileds: "Name", "Email" and "Date of birth")
/users/:Username/FavoriteMovies/:MovieID POST Allows user to add favorite movies.
/users/users/:Username PUT Allows users to update their info
/users/:Username/FavoriteMovies/:MovieID DELETE Allows users to remove a movie from their list of favorites
/users/:Username DELETE Allows a user to delete account
